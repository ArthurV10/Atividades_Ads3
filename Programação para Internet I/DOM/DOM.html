Questão 5

Qual a diferença entre as propriedades textContent, innerText e innerHTML dos elementos HTML? Cite exemplos.

textContent

O que faz:

Trabalha com todo o texto que existe dentro do elemento, inclusive partes que não estão visíveis ou que vêm de elementos filhos. Não interpreta nem executa nenhuma tag HTML — trata tudo como texto “puro”

Exemplo(Coletar estatísticas de conteúdo):

Se você estiver criando uma ferramenta de acessibilidade que mede quantos caracteres um artigo possui, usaria textContent para garantir que conta cada palavra, até mesmo aquelas em parágrafos ocultos ou dentro de <"span"> escondidos.

innerContent

O que faz:

Retorna só o texto que está visível para o usuário, obedecendo CSS como display: none ou visibility: hidden, e respeita quebras de linha causadas por estilos. Também não interpreta tags HTML — tudo vira texto simples.

Exemplo(Gerar relatórios de impressão):

Em um botão “Imprimir página”, coletar somente o que vai aparecer no papel, excluindo menus ou anúncios escondidos por CSS.

innerHTML

O que faz:

Lê ou altera o HTML completo dentro do elemento — ou seja, interpreta tags, atributos e monta elementos DOM. Permite injetar listas, links, imagens, etc., diretamente como marcação.

Exemplo(Inserir snippets de anúncio/marketing):

Carregar banners ou vídeos disponibilizados em HTML por terceiros (sempre validando sanitização).
